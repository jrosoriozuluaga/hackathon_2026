# Checkpoint H4 — Gate de Corte

## Criterios MUST-HAVE (si falla alguno, cortar features)

- [ ] **Pipeline end-to-end funciona**: scrape → normalize → store → score
- [ ] **>= 50 empresas** en Supabase con datos limpios
- [ ] **>= 30 empresas** con signal_score asignado
- [ ] **Dashboard muestra datos** reales (no mocks)
- [ ] **Al menos 1 tipo de signal** detectado (hiring, funding, o press)

## Criterios NICE-TO-HAVE (solo si MUST-HAVE están verdes)

- [ ] Outreach drafts generados para top 10
- [ ] CRM push a Monday funciona
- [ ] 3 flows de Make operativos
- [ ] UI responsive y pulida
- [ ] Comparación side-by-side con Apollo

## Decisión

| Estado de MUST-HAVE | Acción |
|---|---|
| 5/5 verdes | Seguir con plan completo H4–H8 |
| 4/5 verdes | Arreglar el que falta, pausar NICE-TO-HAVE |
| 3/5 o menos | **Modo emergencia**: todo el equipo en pipeline + datos. Cancelar CRM push, outreach, UI fancy |

## Quién decide
Tech Lead tiene veto final. No es democrático — la data manda.
