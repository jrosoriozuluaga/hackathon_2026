# Roadmap

## Phase 1: Subagent Foundation

- Canonical Architect, Frontend, Backend, and Tester subagents.
- Runtime symlink views for Claude and Codex.
- Task, changelog, decision, and roadmap tracking.
- Validation script for repository structure.
- Local artifact workflow that avoids dependency on a specific Harness tool server.

## Phase 2: Dry-Run Harness Scenarios

- React frontend CI/CD dry run.
- Backend Kubernetes deployment dry run.
- Failed pipeline diagnosis dry run.
- New microservice onboarding dry run.

## Phase 3: Expanded Runtime Views

- Add Cursor and Windsurf views when required by active consumers.
- Add runtime-specific permission notes without duplicating canonical prompts.

## Phase 4: Operational Hardening

- Add schema-aware checks around common Harness resource payloads.
- Add sample handoff artifacts for each subagent.
- Track completion rate, retry count, and validation failures per scenario.
